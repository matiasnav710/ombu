Panel UI Demo

Panel UI is a web framework that enables the display of content in a modal sidebar (panel) or a split view. Panel UI is easy to useâ€”just add a data attribute to regular links. This makes it simple to transform standard websites into dynamic user interfaces, supporting tasks that require single views, multi-step workflows, or side-by-side comparisons. Panel UI helps provide an intuitive user experience for complex tasks.

Containers

In the Panel UI framework, there are three content containers.

Full Screen

Data attribute: none

In Panel UI, the default browser display of content is referred to as Full Screen. Links with no data attribute retain the default browser behavior of the new content fully replacing the previous content. Panel UI offers multiple ways a user can transition the view of content from a Panel or Split View to Full Screen.

Use Case

  • Links within text, e.g. help text, paragraph text
  • Pointers to other web pages
  • Pointers to views that are lists of objects

Panel

Data attribute: data-panel-open

Links with the attribute data-panel-open present their content in a modal sidebar, or panel, that slides in from the right of the current view. While a panel is shown the background content is not interactive but remains visible to maintain the context of the current task. An open panel can be expanded into a Full Screen view by selecting the Full Screen action in the panel header.

Use Case

The best use for a panel is to offer quick looks at data and allow the user to perform simple tasks like object creation.

  • Pointers to objects
  • Object creation actions
  • Object editing actions

Split View

Data attribute: data-splitview-open

Links with the attribute data-splitview-open cause the view to be split in half, with the current content being presented on the left and the new content on the right. Both sides of the screen are interactive. In a split view, all links behave like the browser default, no matter the data attribute associated with them.

Use Case

The best use for a split view is for complex tasks that involve comparing or translating data from one view to another.

  • Tabs (Split View enables users to easily compare the content of two tabs side-by-side)
